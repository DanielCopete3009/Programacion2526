Teoría de Java – Programación Orientada a Objetos (POO)

I. Introducción a la Programación Orientada a Objetos

La Programación Orientada a Objetos (POO) es un paradigma de programación que organiza el software en objetos, los cuales representan entidades del mundo real.

En Java, todo gira en torno a las clases y los objetos. Este enfoque facilita:

  • La reutilización del código
  • El mantenimiento del software
  • La escalabilidad de los programas

II. Clases y Objetos

¿Qué es una Clase?

Una clase es una plantilla o molde que define las características y comportamientos de un objeto. 

class Persona {
    String nombre;
    int edad;
}

¿Qué es un Objeto?

Un objeto es una instancia de una clase. Representa un elemento concreto creado a partir de esa plantilla. 

Persona p1 = new Persona();
p1.nombre = "Ana";
p1.edad = 20;

III. Atributos y Métodos

Atributos

Los atributos son variables que describen el estado de un objeto.

Métodos

Los métodos son funciones que definen el comportamiento de un objeto. 

class Persona {
    String nombre;
    int edad;

    void saludar() {
        System.out.println("Hola, me llamo " + nombre);
    }
}

IV. Constructores

Un constructor es un método especial que se ejecuta cuando se crea un objeto. Su función principal es inicializar los atributos. 

class Persona {
    String nombre;
    int edad;

    Persona(String nombre, int edad) {
        this.nombre = nombre;
        this.edad = edad;
    }
}
Persona p1 = new Persona("Carlos", 25);

V. Encapsulación

La encapsulación consiste en ocultar los datos internos de un objeto y permitir el acceso solo mediante métodos públicos.

Modificadores de Acceso

  • private: solo accesible desde la misma clase
  • public: accesible desde cualquier clase
  • protected: accesible desde clases hijas
class Persona {
    private String nombre;

    public String getNombre() {
        return nombre;
    }

    public void setNombre(String nombre) {
        this.nombre = nombre;
    }
}

VI. Herencia

La herencia permite que una clase herede atributos y métodos de otra. Se utiliza la palabra clave extends. 

class Empleado extends Persona {
    double salario;
}

La clase Empleado hereda los atributos y métodos de Persona.

VII. Polimorfismo

El polimorfismo permite que un mismo método se comporte de diferentes formas según el objeto que lo invoque. 

class Animal {
    void hacerSonido() {
        System.out.println("El animal hace un sonido");
    }
}

class Perro extends Animal {
    void hacerSonido() {
        System.out.println("El perro ladra");
    }
}
Animal a = new Perro();
a.hacerSonido(); // El perro ladra

VIII. Abstracción

La abstracción consiste en mostrar solo los detalles esenciales y ocultar la implementación.

Clases Abstractas

abstract class Figura {
    abstract double calcularArea();
}

Interfaces

interface Volable {
    void volar();
}

IX. Resumen de los Pilares de la POO

  • Encapsulación: proteger los datos
  • Herencia: reutilizar código
  • Polimorfismo: múltiples comportamientos
  • Abstracción: ocultar detalles internos
